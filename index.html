About This Tool

This interactive converter helps you visualize how floating-point numbers are represented at the binary level. This converter follows the IEEE 754 floating-point standard and supports standard formats (FP64, FP32, FP16) as well as specialized formats used in machine learning and AI (BF16, TF32, FP8 E4M3, FP8 E5M2). Custom formats can be used as well by specifying the number of sign, exponent, and mantissa bits.

How to Use

1. Choose Your Input Format

Select from standard formats (FP32, FP16, BF16, etc.) or customize the number of sign, exponent, and mantissa bits.

2. Enter a Value

Type any decimal number, or use the value presets to explore special cases like infinity, NaN, or the smallest/largest representable numbers. You can also edit the binary or hexadecimal representation directly.

3. Select Output Format

Choose a different format to see how your number would be represented. The tool automatically shows precision loss when converting between formats.

4. Explore Components

View the breakdown of sign bit, exponent, and mantissa. Interactive binary checkboxes let you flip individual bits to see how they affect the final value.

Common Use Cases

  • ML Engineers: Understand how BF16 and FP16 affect model precision compared to FP32
  • Students: Learn how the IEEE 754 standard works by visualizing the binary representation
  • Developers: Debug floating-point precision issues by examining exact bit patterns
  • Hardware Engineers: Design and validate custom floating-point formats for specialized applications

Supported Formats

  • Standard IEEE 754: FP32 (Single Precision), FP16 (Half Precision), FP64 (Double Precision)
  • Machine Learning Formats: BF16 (Brain Float 16), TF32 (TensorFloat-32), FP8 E4M3, FP8 E5M2
  • Custom Formats: Create any format with 0-15 exponent bits and 0-112 mantissa bits (including fixed-point formats)
  • Microscaling Formats: Small formats like FP6 and FP4 are supported, though some OCP MX Specification encodings that drop NaN/Infinity are not modeled.
